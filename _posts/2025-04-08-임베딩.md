# 임베딩 (Embedding)

> 자연어를 벡터로 바꾸는 결과 또는 과정 전체

---

## 임베딩 방식 분류

### 통계 기반 임베딩
- 단어-문서 행렬을 SVD로 분해
- $A = UDV^\top$ (m×n = m×n · n×n · n×m)

### NN 기반 임베딩
- 신경망을 학습시켜 임베딩 벡터를 획득
- 단어 수준: Word2Vec
- 문장 수준: ELMo, BERT, GPT

---

## 임베딩 철학 3가지

### 1. Bag of Words
- "어떤 단어가 많이 쓰였는가?" (빈도 기반)
- Term-Document Matrix, TF-IDF
- **단어의 순서(Order) 정보를 무시**
- 순서가 다른 단어 나열도 동일한 Document로 취급

### 2. 언어 모델 (Language Model)
- "단어가 어떤 순서로 쓰였는가?"
- 단어 시퀀스에 확률을 부여하는 모델

| 종류 | 설명 |
|---|---|
| 통계 기반 (n-gram) | n개 단어가 동시에 나타날 확률 반환 |
| NN 기반 - ELMo, GPT | 주어진 단어 시퀀스로 다음 단어 예측 |
| 마스크 언어 모델 - BERT | 문장 중간을 마스킹, 해당 위치의 단어 예측 → Bidirectional 학습 가능 |

### 3. 분포 가정 (Distributional Hypothesis)
- "어떤 단어가 같이 쓰였는가?"
- context: 주목하는 단어 주변에 놓인 단어
- 이웃한 단어의 분포를 보고 단어의 의미를 예측

---

## PMI (Pointwise Mutual Information)

> 점별 상호정보량: 두 확률변수 사이 상관성을 계량화

- 두 변수가 완전히 독립이면 PMI = 0
- 단어 A의 등장이 단어 B의 등장 확률에 영향을 줄수록 PMI 증가

---

## Word-Context Matrix

- 주변 단어의 도수를 표시하는 행렬
- 행 벡터 자체를 임베딩으로 사용 가능
- PMI 행렬로 변환하여 임베딩값으로 활용 가능

---

## Pre-train / Fine-tune

| 단계 | 설명 |
|---|---|
| Pre-train (Upstream Task) | 대규모 말뭉치로 임베딩 생성 |
| Fine-tune | 소규모 데이터에 맞게 임베딩 포함 모델 전체를 업데이트 |
| Downstream Task | 품사 판별, 개체명 인식, 의미 분석 등 구체적 문제 |

---

## 단어 수준 vs 문장 수준

| | 단어 수준 | 문장 수준 |
|---|---|---|
| 대표 모델 | Word2Vec | ELMo, BERT, GPT |
| 문제점 | 동음이의어 구분 어려움 | 전이학습 효과 좋음 |
