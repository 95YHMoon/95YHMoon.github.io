# Variational AutoEncoder (VAE)

> AE 기법을 사용하지만, 생성(GAN) 모델에 더 가까운 구조.
> 목적: **실제 데이터 x의 분포를 모방**하는 것

---

## AE와의 차이

| | AE | VAE |
|---|---|---|
| 잠재 표현 | 결정적 벡터 $z$ | 분포에서 샘플링한 $z$ |
| $\mu$, $\sigma$ 노드 | Fully-connected | 분포 내에서 랜덤 샘플링 ($\tilde{z}$) |
| 목적 | 표현 학습 | 생성 |

---

## VAE 구조

$$x \rightarrow \text{Encoder}\; q_\theta(z \mid x) \rightarrow (\mu,\, \sigma) \rightarrow z \sim \mathcal{N}(\mu, \sigma^2) \rightarrow \text{Decoder}\; p_\theta(x \mid z) \rightarrow x'$$

- 인코더가 $\mu$와 $\sigma$를 추정하는 노드를 출력
- 그 분포 안에서 $z$를 랜덤 샘플링
- 학습 완료 후 **Decoder만 떼어 생성 모델로 활용 가능**

---

## 확률밀도함수와 Latent Distribution

- $X \sim \mathcal{N}(\mu, \sigma^2)$
- $h_1$, $h_2$ 노드가 각각 $\mu$, $\sigma$에 해당하는 분포를 출력
- $\mu_1$, $\sigma_1$을 알면 분포 전체를 안다는 의미

---

## Reparameterization Trick

### 문제
- $z$를 분포에서 랜덤 샘플링하면 backpropagation이 불가
  - $z$가 확률적 노드이므로 $\mu$, $\sigma$에 대해 미분 불가

### 해결
- $z$를 직접 샘플링하지 않고, 표준 정규분포 $\epsilon$을 사용

$$z = \mu + \sigma \cdot \epsilon, \qquad \epsilon \sim \mathcal{N}(0,\, 1)$$

- $z \sim \mathcal{N}(170,\, 5^2)$에서 $z=180$을 뽑는 것과
- $\epsilon \sim \mathcal{N}(0,1)$에서 $\epsilon=2$를 뽑아 $z = 170 + 5 \times 2 = 180$을 만드는 것은 **동일**

### 미분 가능 이유

$$\frac{\partial z}{\partial \mu} = 1, \qquad \frac{\partial z}{\partial \sigma} = \epsilon$$

- $\epsilon$은 외부에서 뽑힌 상수 → $\mu$, $\sigma$에 대해 미분 가능

### log sigma 처리
- $\sigma$가 음수가 나오는 경우 방지를 위해 $\log\sigma$를 출력

$$z = \mu + e^{\log\sigma} \cdot \epsilon$$

---

## VAE Loss Function

$$\mathcal{L} = \underbrace{-\mathbb{E}_{q(z \mid x)}\!\left[\log p(x \mid z)\right]}_{\text{Reconstruction Error}} + \underbrace{KL\!\left(q(z \mid x) \;\|\; p(z)\right)}_{\text{Regularization}}$$

### Reconstruction Error

$$-\mathbb{E}_{q(z \mid x)}\!\left[\log p(x \mid z)\right]$$

- $x$가 주어졌을 때 $z$ 분포로부터 나온 기댓값
- 입력과 출력 간의 복원 오류

### Regularization: KL Divergence

$$KL\!\left(q(z \mid x) \;\|\; p(z)\right)$$

- $q(z \mid x)$: $x$를 넣어 나온 분포
- $p(z)$: 내가 정해준 표준 정규분포 $\mathcal{N}(0,1)$
- 두 분포가 같아질수록 KL값 → 0
- **VAE가 AE와 달리 생성 모델에 활용되는 이유**: KL Divergence의 존재로 인해 $z$ 공간이 $\mathcal{N}(0,1)$에 가깝게 정규화됨

---

## VAE가 생성 모델로 활용되는 이유

- 학습 완료 후 Decoder만 남긴 상태에서
- $\mathcal{N}(0,1)$에서 새로운 $z$를 샘플링하면
- 계속 다르지만 비슷한 새로운 $x$가 생성됨

---

## CVAE (Conditional VAE)

- 레이블 $y$ 정보가 있을 경우 VAE를 조건부로 수행
- Vanilla VAE에 $y$ 조건 추가
- 예: MNIST에서 $x$ + label 값을 함께 입력 → VAE보다 빠르게 수렴
