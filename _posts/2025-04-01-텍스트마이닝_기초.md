# 텍스트 마이닝 기초

> 비정형 데이터(텍스트) → 정형 데이터로 변환하는 과정
> 데이터 마이닝 + NLP의 결합

---

## 기본 용어

| 용어 | 설명 |
|---|---|
| Token | 분리자를 포함하지 않는 연속적 문자열 (단어, 형태소, 서브워드) |
| Term | 특정 의미를 갖는 토큰 (의미 없는 단어 제외) |
| Document | 문서 |
| Corpus | Document의 집합, 말뭉치 |
| Vocabulary | Corpus의 문서를 토크나이즈 후 중복 제거한 토큰들의 집합 |
| Unknown Word | 어휘 집합에 없는 토큰 (신조어 등) |

계층 관계: `Corpus > Document > Token & Term`

---

## 전처리 과정

### Tokenize (토크나이즈)
- 문장을 Token 단위로 분리
- 도구: mecab, kkma

### Tagging
- 각 토큰에 품사 태그 부착
- ex) `school/Noun`

### 형태소 분석
- Tokenize + Tagging을 합친 것

### Stemming (어근 분리)
- 단어를 어근으로 통합
- ex) `affectation`, `affected`, `affecting` → `affect`
- 역할: 단어 차원을 컴팩트하게 압축

### Stopwords
- 흔하고 의미 없는 단어 제거
- ex) `the`, `is`, `a`

---

## Term-Document Matrix

- 문서에서 각 용어가 나타나는 횟수를 행렬로 표현
- 문제점:
  - 0이 매우 많음 (희소 행렬)
  - 단어 차원이 매우 큼

---

## TF-IDF

> 단어의 중요도를 수치화하는 방법

### TF (Term Frequency)
- 한 문서 내 단어 빈도
- `log(1 + f_ij)`

### IDF (Inverse Document Frequency)
- 여러 문서에 걸쳐 나타나는 단어의 가중치를 낮춤
- `log_2(N / d_j)`
  - N: 전체 문서 수
  - d_j: 해당 단어가 나타난 문서 수

### 해석
- 한 문서에서 빈도가 높은 단어 → 높은 가중치 (문서 특성 반영)
- 많은 문서에 흔히 나타나는 단어 → 낮은 가중치
